Функции

Что такое функции?

Условно, функцию можно представить как именованный фрагмент программного кода, который можно вызывать многократно из разных частей программы. Она не просто хранит данные, а выполняет определённую последовательность действий.

Зачем они нужны?

С функциями программы становятся более модульными, а процесс программирования значительно упрощается. Они позволяют избежать дублирования кода, что является ключевым принципом хорошего программирования. Этот принцип называется DRY – Don’t Repeat Yourself (Не повторяйся). Как только вы замечаете в программе повторение (дублирование) кода, это сигнал о том, что архитектуру следует пересмотреть. Повторяющийся фрагмент можно поместить в функцию, а затем вызывать ее в нужных местах программы. Это основная причина введения функций в языки программирования.

Основы создания функций

Чтобы создать функцию в JS, используется ключевое слово function, за которым следует имя функции, параметры в скобках и тело функции в фигурных скобках, где мы пишем, что она должна делать, а также вызов функции по имени:

function greet() {
console.log('Hello, world!');
}
greet(); //вывод 'hello, world!'

Параметры и возвращаемые значения

Функции могут принимать параметры и возвращать значения. Можно передавать сразу несколько параметров через запятую. 

function add(a, b) {
return a + b;
}
const result = add(5, 3);
console.log(result); //вывод 8

Здесь функция add принимает два параметра и возращает их сумму. Использование return позволяет вернуть результат работы функции.

Стрелочные функции

Стрелочные функции - это сокращенный синтаксис для создания функций, особенно полезный при использовании функции обратного вызова. Главное отличие - вместо function используется стрелка =>. Еще одна особенность: если внутри функции всего одна строка, можно не писать фигурные скобки и return - JS все поймет сам.

const subtract = (a, b) => a - b;
console.log(subtract(10, 3)); //Вывод 7.

Преимущества стрелочных функций в том, что меньше кода, проще читать, нет лишних скобок, все ясно с первого взгляда. Если тело функции состоит из одного выражения, то оператор return можно опустить.

Функции высшего порядка

Функции, которые принимают другие функции в качестве параметров или возвращают фукции, называют функциями высшего порядка.

function applyOperation(a, b, operation) {
return operation(a, b);
}
const result1 = applyOperation(5, 10, (x, y) => x + y);
console.log(result1); //вывод 15
const result2 = applyOperation(10, 3, (x, y) => x * y);
console.log(result2); //вывод 30

Функция applyOperation принимает числа a, b и функцию operation, которая определяет, что делать с этими числами. Это мощный способ сделать код гибким и легко настраиваемым.

Рекурсивные функции

Функция, которая вызывает сама себя. Полезна для решения задач, которые можно разделить на более простые задачи.

function factorial(n) {
if (n == 1) return 1;
return n * factorial(n - 1);
}
console.log(factorial(5)); //выход 120

В этом примере функция рекурсивно вызывает сама себя для вычисления факториала числа n.

Замыкания

Это мощная функция, которая позволяет функции запомнить свое лексическое окружение даже после того, как она была вызвана.

function createCounter() {
let count = 0;
return function() {
count++;
return count;
};
}
const counter = createCounter();
console.log(counter()); //вывод 1
console.log(counter()); //вывод 2

Функция createCounter возвращает внутреннюю функцию, которая имеет доступ к переменной count, даже после завершения выполнения createCounter.

Пример сложной функции: каррирование

Каррирование - это техника, когда функция разбивается на несколько функций, каждая из которых принимает один аргумент.

function multiply(a) {
return function(b) {
return function(c) {
return a * b * c;
};
};
}
console.log(multiply(2)(3)(4)); //вывод 24

Такой подход полезен, когда нужно постепенно применять аргументы к функции.

const getRandomColor = () =>

2. Каррированная функция для умножения

const multiply = a => b => a * b;

// Использование
const double = multiply(2);
const triple = multiply(3);

console.log(double(5));  // 10
console.log(triple(5));  // 15

3. Функция - валидатор email с замыканием

const createEmailValidator = () => {
  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return email =>emailRegex.test(email);
};

// Использование
const isValidEmail = createEmailValidator();
console.log(isValidEmail('test@example.com'));  // true
console.log(isValidEmail('invalid-email'));     // false

4. Функция - декоратор для замера времени выполнения

const measureTime = (fn) => (...args) => {
  const start = performance.now();
  const result = fn(...args);
  const end = performance.now();
console.log(`Времявыполнения: ${(end - start).toFixed(2)} мс`);
  return result;
};

// Использование
const slowFunction = measureTime((n) => {
  let sum = 0;
  for (let i = 0; i< n; i++) sum += i;
returnsum;

slowFunction(1000000); // Выведет время выполнения и вернет результат

5. Функция для создания генератора последовательностей

const createSequence = (start = 0, step = 1) => {
  let current = start;
  return () => {
    const value = current;
    current += step;
    return value;
  };
};

// Использование
const nextId = createSequence(100, 5);
console.log(nextId()); // 100
console.log(nextId()); // 105
console.log(nextId()); // 110
