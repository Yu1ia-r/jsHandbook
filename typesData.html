Типы данных

JavaScript — язык с динамической типизацией. Это означает, что при объявлении переменной мы не указываем её тип явно, и тип значения может изменяться во время работы программы. Для эффективной работы и понимания кода важно разбираться какие типы данных существуют в языке, как они работают и где применяются.

Number (числа)

В JavaScript все числа представлены в виде 64-битных чисел с плавающей точкой (формат IEEE 754). Числа могут быть целыми и дробными. 

let a = 10;  // целое число
let b = 3.14;  // дробное число       
let c = -100;  // отрицательное число       

alert(1 / 0); // Infinity  
alert(-1 / 0);  //-Infinity
alert("abc" / 3); //NaN

Особенности:

Операции с числами довольно просты, но из-за плавающей точки могут возникать неточные вычисления: 

alert(0.1 + 0.2); //0.300000000000004(ошибка округления)

BigInt (большие целые числа)

BigInt позволяет работать с очень большими целыми числами, которые не помещаются в Number (максимальное значение ~9 квинтиллионов). Для создания BigInt добавляется суффикс n: 

let big = 900719925474099199999n;

alert (big + 1n);     // 900719925474099200000n
alert(typeof big);   // "bigint"

String (строки)

Строка — это последовательность символов, заключенная в кавычки. Можно использовать: Двойные " " Одинарные ' ' Обратные кавычки `, которые позволяют создавать многострочные строки и вставлять выражения (template literals).

let str1 = "Привет";
let str2 = 'Мир';
let str3 = `Сумма: ${a + b}`;

alert(str1 + " " + str2); //”Привет Мир”  
alert(str3);  //Сумма:13.14

Boolean (логический тип)

    Логический тип имеет два возможных значения:
  • true — истина
  • false — ложь

Он широко используется в условиях и циклах.Они используются для таких вещей, где ответы предполагают два варианта ответа - да или нет.

let a = true; 
let b = false; 

alert(a && b);//false (логическое И)
alert(a || b);//true(логическое ИЛИ)  
alert(!a); //false(отрицание)

Undefined (неопределённый тип)

undefined — автоматическое значение переменной, если ей не присвоено никакое значение.

let x;
alert(x);  //undefined            
alert(typeof x);  //”undefined”

Можно явно присвоить undefined, но делать это не рекомендуется — лучше использовать null (см. далее).

Null (пустое значение)

null — специальное значение, указывающее, что значение отсутствует.

let y = null;
alert(y);  //null
alert(typeof y); //”object”-историческая особенность JS, на самом деле null не объект

null часто используется, чтобы явно указать, что переменная "пустая" или "очищена".

Symbol (символы)

Symbol — примитивный тип, появившийся в ECMAScript 2015 (ES6). Он создаёт уникальные идентификаторы, которые нельзя случайно перезаписать или столкнуть с другими именами. В основном использующееся для отладки кода.

let sym1 = Symbol("id");
let sym2 = Symbol("id");

alert(sym1 === sym2);  //false- даже с одинаковыми описаниями они уникальны

Принципы работы с типами данных

JavaScript автоматически преобразует (кастует) типы в определённых операциях — это называется неявным приведением типов:

alert("5" - 2);    // 3 (строка превратилась в число)
alert("5" + 2);    // "52" (число превратилась в строку)
alert(Boolean("")); // false (пустая строка — «ложное» значение)

Тем не менее, смешанные типы могут приводить к трудноуловимым багам, поэтому лучше использовать явное преобразование:

let a = "5";
let b = Number(a) + 2;  // 7
